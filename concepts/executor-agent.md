# Executor Agent

The agent responsible for solving tasks generated by the [[curriculum-agent]]. In [[agent0-framework]], it is trained via [[adpo]] (Ambiguity-Dynamic Policy Optimization) with multi-turn tool integration.

## Characteristics

- Receives tasks from the curriculum agent
- Can invoke external tools (e.g., Python interpreter) through multi-turn interaction
- Trained with RL on self-generated solutions using majority voting as pseudo-labels

## Training Challenge

Majority voting produces noisy pseudo-labels. When consistency is low (high ambiguity), the label is unreliable. [[adpo]] solves this by dynamically adjusting the trust region based on ambiguity.
